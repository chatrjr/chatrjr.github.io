    Been a while, eh? I spent the last week and a half building version 3 of this humble site. As reliable as Jekyll has been, I wanted to give v3 more than a fresh coat of paint. I’d been fidgeting to go back under the hood and replace the engine.

    §What is Hexo?

    Hexo is a robust static site generator built on Node.js. Part of the reason I was drawn to it after looking around is that it’s pretty well documented. While clear, a lot of the documentation isn’t in context. Part of the reason for this post is to help others get up and running with this platform quickly. Especially because I built my theme from scratch.

    §Better Than Jekyll?

    That’s not the right question to ask. Jekyll is an incredibly well-documented, battle-ready blogging platform that was a breeze to get started with and has a “set and forget” integration with GitHub Pages. The advantages that pulled me over to Hexo was not a matter of it being objectively better. It just meets my particular needs better.

    §Getting to Know Hexo

    For the purpose of this post I won’t be reviewing this blog’s source. As a theme built from scratch, it doesn’t follow many of the API features of Hexo. So instead, we’ll start from Hexo’s defaults. You’re still welcome to view the source after it’s clear what we’re doing here.

    §Setting Up

    I’m gonna condense the set up process, because that’s not our focus today. Fire up that terminal. Make sure you have Node.js and Git installed before we begin.

    §1. Grab the Hexo CLI

    Assuming you’ve fulfilled the above conditions, put in your terminal:

    npm install hexo-cli -g

    That will install Hexo’s command line interface, and then you’ll be ready for the next step.

    §2. Initialize a New Hexo Site

    Now, cd to the directory you want to build out your site. Now enter in your terminal:

    hexo init hexo-tutorial
cd hexo-tutorial

    You’ll see that Hexo has built out a scaffold that resembles this directory tree:

    §The Barest of Minimums to Blog with Hexo

    §Writing a New Post

    If you want to get down to blogging immediately, you can enter in your terminal at this point:

    hexo new post "Title of Post"

    This will create a new post under source/_posts built out from the scaffolds/post.md file and its metadata.

    You can also write drafts by creating source/_drafts and then making a new draft with:

    hexo new draft "Title of Pending Post"

    And then publish your post when you’re ready.

    hexo publish post "Title of Finished Post"

    §Installing a Deploy Plugin

    The lowest friction way for a developer to blog is with GitHub Pages, and Hexo has a convenient deployment method that leverages Git for such an occasion.

    Start by installing the module:

    npm install hexo-deployer-git --save

    Now in the configuration where it’s marked deploy:, modify it to the following:

    # Deployment
## Docs: http://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: https://github.com/username/username.github.io.git
  branch: master

    Make sure there’s already a .nojekyll file (and optionally place a CNAME file with your custom domain in the source folder) in the target repo before the next step.

    No, really. Don't neglect this part. If you don't put that file in the repo, GitHub Pages will attempt to build your site as a Jekyll site even though it isn't one. This will cause it to build well enough initially, but prevent any updates. Which is... a problem.

    §Hello, World

    When you’re ready to publish your site, crack open your terminal and issue the following commands:

    hexo generate
hexo deploy

    You can also deploy in one command with the -d flag on hexo generate.

    Hexo will ask for your GitHub username and password (unless you have SSH access) and your new site will be live.

    Excellent! Now get to writing, you’re done with the hard part. Content rules the web, and landscape is great a default theme, so there’s no harm in stopping here.

    However, if you’re curious to know how you can bend Hexo to your will, then strap in for the ride.

    §Now for Some Non-Euclidean Geometry

    If you’re still with me, let’s start tweaking some settings. First thing: we’ll want to configure _config.yml so there are no collisions with Hexo’s development server.

    # Extensions
## Plugins: http://hexo.io/plugins/
## Themes: http://hexo.io/themes/

server:
  port: 3021
  log: false

theme: landscape

    Now with that out of the way, let’s start some in-depth customization.
